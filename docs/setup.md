# Setup Guide

This document explains how to set up EngageFlow for local development.

---

## Prerequisites

The host machine only needs:

- [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- [Git](https://git-scm.com/)
- An editor (e.g. [VS Code](https://code.visualstudio.com/))

PHP, Composer, Node, npm, and MySQL do **not** need to be installed on the host machine. All runtimes run inside Docker containers.

---

## Container-First Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/shahrilazwa/engageflow.git
cd engageflow

# 2. Copy the environment file
cp .env.example .env

# 3. Start the database first so the app container can connect
docker compose up -d db

# 4. Generate APP_KEY — this writes the key into .env on the host
docker compose run --rm app php artisan key:generate

# 5. Start all services (force-recreate app so it picks up the new APP_KEY)
docker compose up -d --force-recreate app node

# 6. Run database migrations
docker compose exec app php artisan migrate

# 7. Open the app
# http://localhost:8000
```

> **Why the force-recreate step?**
> `APP_KEY` is passed into the `app` container as an environment variable at startup.
> If you generate the key after the container is already running, the container will not see the new value until it is recreated.
> Running `docker compose up -d --force-recreate app` restarts only the app container with the updated key.

## Docker Compose Services

| Service | Image | Purpose |
|---|---|---|
| `app` | Custom PHP 8.4 + Nginx | Laravel application server |
| `node` | Node LTS | Frontend asset compilation |
| `db` | MySQL 8 | Database |

Redis and Mailpit are not included in v1.

---

## Running Commands Inside Containers

All Composer and Artisan commands run inside the `app` container:

```bash
# Run Artisan commands
docker compose exec app php artisan <command>

# Run Composer commands
docker compose exec app composer <command>

# Run npm commands (via node container)
docker compose exec node npm <command>
```

---

## Useful Commands

```bash
# View running containers
docker compose ps

# View app logs
docker compose logs app

# Stop all containers
docker compose down

# Rebuild the app image (after Dockerfile changes)
docker compose build --no-cache app

# Run tests
docker compose exec app php artisan test

# Run PHPStan
docker compose exec app vendor/bin/phpstan analyse

# Run Pint (code style check)
docker compose exec app vendor/bin/pint --test

# Run Pint (auto-fix)
docker compose exec app vendor/bin/pint
```

## Environment Variables

Copy `.env.example` to `.env` and review the values. Key variables:

| Variable | Description |
|---|---|
| `APP_KEY` | Generated by `php artisan key:generate` |
| `APP_URL` | `http://localhost:8000` for local development |
| `DB_HOST` | `db` (the Docker Compose service name) |
| `DB_DATABASE` | `engageflow` |
| `DB_USERNAME` | `engageflow` |
| `DB_PASSWORD` | `secret` |
