# Deliverables

This document will describe Task Deliverables once MVP 4 is implemented.

---

## Scope

Deliverables track expected Task outputs separately from workflow progress.

V1 supports:

- Deliverable title
- Deliverable type
- Deliverable status
- Optional description
- Optional due date
- Remarks
- Overdue display
- Project-scoped access through the parent Task

V1 does not include file upload or external repository integrations.

---

## Relationship To Tasks

Deliverables belong to Tasks. They do not replace TaskWorkflowStep progress and do not change the workflow progress percentage.

---

## Document Links

Deliverables may have external Document Links after DocumentLink support is implemented. Document Links store URLs only.

---

## Review Notes

Deliverable UI should be compact and operational. It should not overwhelm the Task detail workflow timeline.
