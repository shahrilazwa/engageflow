# Setup Guide

This document explains how to set up EngageFlow for local development.

---

## Prerequisites

The host machine only needs:

- [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- [Git](https://git-scm.com/)
- An editor, such as [VS Code](https://code.visualstudio.com/)

PHP, Composer, Node, npm, and PostgreSQL do not need to be installed on the host machine. All runtimes run inside Docker containers.

---

## Container-First Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/shahrilazwa/engageflow.git
cd engageflow

# 2. Copy the environment file
cp .env.example .env

# 3. Start the database first so the app container can connect
docker compose up -d db

# 4. Generate APP_KEY. This writes the key into .env on the host
docker compose run --rm app php artisan key:generate

# 5. Install frontend dependencies inside the node container
docker compose run --rm node npm ci

# 6. Start all services
docker compose up -d --force-recreate app node

# 7. Run database migrations
docker compose exec app php artisan migrate

# 8. Open the app
# http://localhost:8000
```

`APP_KEY` is read by Laravel from the mounted `.env` file. Generate it before opening the app in the browser.

---

## Docker Compose Services

| Service | Image | Purpose |
|---|---|---|
| `app` | Custom PHP 8.4 + Nginx | Laravel application server, Composer, Artisan, Pest, PHPStan, and Pint |
| `node` | Node LTS | npm, TypeScript, Vite, and frontend asset compilation |
| `db` | PostgreSQL 17 | Local and test database |

Redis and Mailpit are not included in v1.

---

## Running Commands Inside Containers

```bash
# Run Artisan commands
docker compose exec app php artisan <command>

# Run Composer commands
docker compose exec app composer <command>

# Run npm commands
docker compose exec node npm <command>
```

Do not run Composer, PHP, Node, or npm commands directly on the host machine. The `vendor` and `node_modules` dependency directories are mounted as Docker named volumes.

---

## Useful Commands

```bash
# View running containers
docker compose ps

# View app logs
docker compose logs app

# Stop all containers
docker compose down

# Rebuild the app image after Dockerfile changes
docker compose build --no-cache app

# Run tests
docker compose exec app php artisan test

# Run PHPStan
docker compose exec app vendor/bin/phpstan analyse

# Run Pint
docker compose exec app vendor/bin/pint --test

# Run TypeScript checks
docker compose exec node npm run typecheck

# Build frontend assets
docker compose exec node npm run build
```

## Environment Variables

Copy `.env.example` to `.env` and review the values.

| Variable | Description |
|---|---|
| `APP_KEY` | Generated by `php artisan key:generate` inside the `app` container |
| `APP_URL` | `http://localhost:8000` for local development |
| `DB_HOST` | `db`, the Docker Compose service name |
| `DB_DATABASE` | `engageflow` |
| `DB_USERNAME` | `engageflow` |
| `DB_PASSWORD` | `secret` |

## Seeded Demo Data

A test user is seeded by default. Run:

```bash
docker compose exec app php artisan migrate:fresh --seed
```

**Test user credentials:**
- Email: `test@example.com`
- Password: `password`

This creates a local user you can log in with immediately. Additional seeders for Projects, workflows, Tasks, deliverables, and follow-up actions will be added in later MVP slices.
